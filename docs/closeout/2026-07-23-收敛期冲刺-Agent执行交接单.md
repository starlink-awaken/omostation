## 1. 执行护栏（RED LINE — 动手前必读）

本仓有强制工作流纪律（ADR-0203），**任何改动必须先 `start` 再改文件**，否则视为违规、门禁会拦。

**每个任务开工前，固定三步：**

```bash
# ① 启动会话上下文
uv run --with "pyyaml" python "bin/agent-workflow.py" bootstrap
uv run --with "pyyaml" python "bin/agent-workflow.py" status --json

# ② 有 diff 时先让工具选对 workflow（防错位）
uv run --with "pyyaml" python "bin/agent-workflow.py" suggest --from-diff --profile <你的-profile>

# ③ start + claim（把要改的路径登记进 run）
uv run --with "pyyaml" python "bin/agent-workflow.py" start <workflow-id> \
  --profile <你的-profile> --objective "<本任务摘要>"
uv run --with "pyyaml" python "bin/agent-workflow.py" claim <run-id> --path <path>
```

**其它硬约束：**

- **不碰主仓 `main`**：所有落地走 `bin/gac/gac-worktree.sh`（claim → submit → merge），不得直接在 main 上写。
- **SSOT 铁律**：运行态读 `.omo/state/system.yaml`；端口读 `protocols/port-registry.yaml`；vault 路径读 `protocols/vault-paths.yaml`；**禁止硬编码**这些值。
- **`.omo/` 治理写**：走 `omo` CLI/MCP 或批准的 broker，不做 ad-hoc 文件 I/O。
- **不 commit / push / reset / 删除**，除非任务步骤明确要求且已获授权。
- **收敛期纪律**：不新增愿景功能（多机/蜂群/角色框架都先别碰），发现诱惑顶住。
- 收尾统一走：`make gac-local-gate` + `uv run --with "pyyaml" python bin/ssot/ssot-guardian.py`。
