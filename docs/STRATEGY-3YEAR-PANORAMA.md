# eCOS v6 · 织星三年综合全景战略规划（2026H2 – 2029）

> **一句话战略**：过去三年我们把「蜂群式 AI 超级大脑」的**图纸**画到了 A 级；
> 未来三年的战略主线，是把**实物**从 B- 级抬到 A 级——不是再叠架构，而是
> **启用已有治本机制、让度量反映执行面、把个人数字大脑真正跑起来**。

---

## 一、执行摘要（TL;DR）

织星（omostation / eCOS v6）已建成一套罕见完整的 AI 操作系统骨架：`5+4+1+1` 分层、
17 个子项目、agora I0 单点 MCP 网关、SSOT 严格的治理即代码（GaC）体系。**设计质量是稀缺资产**。

但 2026-07-14 全盘巡检暴露了系统的**结构性张力**——「声明-执行鸿沟」（decl-exec-gap）：

| 维度 | 声明面（图纸） | 执行面（实物）| 快照 |
|------|--------------|-------------|------|
| 成熟度 / 安全度 | 100 / 100 | — | system.yaml |
| 复合健康分 | — | **83**（警戒）| BRIEF.md |
| L1 runtime daemon 在线率 | — | **~60%** | 架构分析 |
| 多 Agent 协调 | AdvisoryLock + P74 契约齐备 | worktree 隔离**就绪未启用**，实战靠人工等/停 | rollout memory |

**三年战略判断**：最大的杠杆点不在"继续加架构"，而在三件事——
（1）**收敛执行面**：启用 agent isolation、修复 runtime 健康、让健康分反映实物；
（2）**兑现愿景**：把"蜂群/多机/多角色"从蓝图变成个人日常可用的"外置数字大脑"；
（3）**沉淀护城河**：把 GaC + SSOT + MCP 网关这套治理资产产品化，形成可复制的方法论。

对应三个阶段：**2026H2–2027 收敛期（治本）→ 2027–2028 兑现期（蜂群落地）→ 2028–2029 跃迁期（个人大脑 + 外溢）**。

## 二、现状诊断（Where We Are）

### 2.1 分层健康度矩阵（截至 2026-07-14 巡检）

```
L4 文档层     A    doc-ssot-contract 严格，19 类 SSOT 各有权威源
L3 入口层     A    agora MCP gateway 统一 + cockpit 单一人类入口
L2 内核层     A-   kairon God Module 全拆，16 包 ~155K LOC 稳
I0 织层       A    agora hub 单点接入，命名空间内置，114 BOS 服务
X1–X4 治理    B+   GaC 强，但 worktree 假阳性 + agent 协调弱
L1 运行时     C+   ⚠️ daemon ~60%，ollama stale，探测假阳性
L0 协议层     A    ecos MOF M1–M3 + L0 constraints 稳
```

**最弱环节 = L1 运行时**。这是战略的第一优先修复对象。

### 2.2 三大架构支柱

- ✅ **支柱一 · MCP Gateway（已闭环）**：agora I0 hub + 14 mcp-server 声明式注册，
  FastMCP 统一、`service.tool` 命名空间内置。AI agent 单点接入是 eCOS 最完整的成果。
- ⚠️ **支柱二 · Submodule 治理（脆弱→已补巡检）**：17 submodule，2026-07-14 实战出现
  gitlink 全盘分叉，已用 `submodule-gitlink-check.py` 治本，待挂 6h cron。
- 🔴 **支柱三 · 多 Agent 协调（最大未解痛点）**：隔离机制**就绪未启用**，并发 agent 抢主仓
  main，人工无法秒级介入。**这是三年战略中最高优先级的"拍板"决策。**

### 2.3 核心矛盾：声明-执行鸿沟（元模式）

`decl-exec-gap-meta-pattern`（memory 已沉淀 11+ 实例）：maturity/safety 全 100（图纸漂亮），
但 health 83 / daemon 60% / 并发 agent 造成 main 混乱（实物粗糙）。**架构图纸质量远超运行质量。**
一切战略举措都应服务于"缩小这条鸿沟"，而非扩大图纸领先优势。

## 三、三年愿景（Where We Are Going）

### 3.1 终极愿景（不变）

> **蜂群式 AI 超级大脑 · 个人 OPC 外置数字大脑**
> 多机协作 · 多角色 Agent · 多场景支持 · 多领域融合

### 3.2 三年后（2029）的具体样子

| 能力 | 2026 现状 | 2029 目标态 |
|------|----------|------------|
| **多机协作** | 单机为主，本地协作 | 3–5 台设备（工作站/笔电/手机）无缝共享同一数字大脑 |
| **多角色 Agent** | 声明面契约齐备，执行面靠人工协调 | 角色框架标准化，多 agent 并发**互不破坏**、自动闭环 |
| **蜂群智能** | 蓝图阶段 | 涌现检测 + 集体决策可用于真实决策辅助 |
| **个人数字大脑** | 知识/记忆基座（gbrain/kos）已立 | 个人知识图谱 + 偏好学习 + 数字孪生，日常高频使用 |
| **运行健康** | health 83 / daemon 60% | health ≥ 95（执行面口径）/ daemon ≥ 99% |
| **治理资产** | 内部自用 | GaC+SSOT+MCP 方法论产品化，可对外复制 |

### 3.3 战略主题（贯穿三年的三条主线）

1. **执行面第一（Runtime-First）**：运行事实是第一公民，度量必须反映实物而非图纸。
2. **蜂群兑现（Swarm-Realization）**：从"能画"到"能跑"，多机多角色进入日常。
3. **个人专属（Personal-by-Design）**：一切能力最终收敛于"我的外置大脑"这一个人价值。

## 四、五大战略支柱（Strategic Pillars）

### 支柱 P1 · 收敛执行面，让健康分反映实物
把"图纸满分、实物警戒"的鸿沟作为头号治理债务。启用 agent isolation、修复 L1 runtime、
重构 health_score 权重（提升 daemon 在线率权重、下调声明面 maturity 权重）。
**成功标准**：health_score 口径切换后仍 ≥ 90，且与 daemon 实测强相关。

### 支柱 P2 · 多 Agent 协调从"人工"到"自治"
落地 worktree + branch protection，强制每 agent 走 `gac-worktree.sh claim`，不碰主仓 main。
P74 沉默检测常态化。**成功标准**：并发 agent 零主仓冲突，人工介入频次趋近于 0。

### 支柱 P3 · 蜂群与多机协作落地
按 VISION-ROADMAP 的 Phase 2–4 推进：Agent 注册中心、分布式调度、角色框架、涌现检测。
但每一步以"执行面可验证"为门禁（2 机延迟 / 4 机成功率 / 3 角色完成率）。
**成功标准**：多机协作从 30% 提升到 ≥ 85% 真实可用。

### 支柱 P4 · 个人数字大脑兑现
以 gbrain + kos 为基座，建个人知识图谱、偏好学习、记忆管理、AI 助手、数字孪生。
把 X3 价值仪表（创意创作/工作交付/知识复用）从"计数"升级为"个人价值闭环"。
**成功标准**：日常高频使用，个人知识图谱覆盖率 ≥ 90%。

### 支柱 P5 · 治理资产产品化（护城河外溢）
GaC + SSOT + MCP 网关 + MOF 元模型是可复制的方法论。三年内沉淀为对外可用的
"AI 操作系统治理框架"，形成技术品牌与潜在商业化路径。
**成功标准**：方法论文档化 + 至少一次对外复用验证。

## 六、度量体系演进（Metrics That Matter）

战略成败取决于**度量口径本身的诚实度**。三年内推动度量从"声明面"迁移到"执行面"：

| 指标 | 现口径 | 目标口径（执行面）|
|------|--------|------------------|
| health_score | 声明面 maturity 主导（83，警戒）| daemon 在线率 + 实测权重主导，≥ 95 |
| daemon 在线率 | ~60%（探测有假阳性）| ≥ 99%，探测去假阳性 |
| 多 Agent 冲突 | 靠人工等/停 | 零主仓冲突，自动闭环 |
| X3 价值仪表 | 创作 636 / 交付 4 / KOS 索引 0（计数）| 升级为"个人价值闭环"质量指标 |
| 多机/多角色/蜂群/个人大脑 | 30% / 40% / 50% / 75% | 85% / 85% / 85% / 90% |

> ⚠️ **KOS 索引篇 = 0** 是一个值得警惕的信号：知识复用维度的"物理数据源"尚未真正积累。
> 个人大脑愿景的兑现，必须先让 KOS 从 0 走向有实质索引量。
