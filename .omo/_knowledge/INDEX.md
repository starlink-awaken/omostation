# Knowledge Plane — 知识面

> 归档计划、复盘、审计、指南、参考。
> 沉淀可复用认知，把治理经验变成组织记忆。

## 核心目录

| 目录 | 职责 |
|------|------|
| `plans/` | 阶段规划与蓝图 |
| `summaries/` | 阶段总结与验收报告 |
| `audits/` | 审计报告 |
| `_knowledge/design/` | 设计文档与架构决策 |
| `_knowledge/management/` | 管理指南与流程 |
| `_knowledge/process/` | 过程复盘与 retro |
| `_knowledge/reference/` | 参考材料 |
| `_knowledge/usage/` | 使用指南 |
| `_knowledge/superpowers/` | 能力目录 |

## 写入规则

- 计划必须回指 control 面的目标与门禁
- 复盘必须回指 delivery 面的证据与测试结果
- 新增知识文档必须通过平面判定：不属于 control/truth/delivery 才落 knowledge
