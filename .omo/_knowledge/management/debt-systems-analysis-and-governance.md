# 债务冰山下行分析 (Iceberg Analysis)

## 事件层 (Layer 1) — 可见的债务事件

1. KOS ruff: 5,263 → 78 (98.5% 清理完成) ➜ 治理有成效但仍有残余
2. Minerva ruff: 955 → 17 (98.2% 清理完成) ➜ 同上
3. Hermes 179 断链 → 已清理，但 3 个 launchd 服务仍处于崩溃状态
4. 2 个敏感能力连接器 (Apple, WeChat/SMB/Media) 已阻塞了 11+ 个 Phase
5. CI E2E 测试环境 (D2) — 跨越了 P5→P16 共 12 个 Phase 仍未解决
6. eu-pricing 独立测试 (D3) — 同样跨越了 12 个 Phase 仍未解决
7. SharedBrain 210 万行零测试 — 从 P2 发现至今从未排期
8. KOS 零消费者 — API 从未被其他项目 import

## 模式层 (Layer 2) — 时间线上的重复模式

**模式 A: "债务迁移"**
- 每次 Phase 规划时，Hard 债务项被标记为"下一阶段处理"
- D2/D3 跨越了 12 个 Phase，形成了结构性拖延
- SharedBrain 零测试从 P2→P16 从未被安排

**模式 B: "Phase 健康分膨胀"**
- P5→P8: 健康分停滞在 90
- P9→P16: 健康分从 90 跃升至 97
- 但核心债务项 (D2, D3, SharedBrain) 并未解决
- 健康分的提升来自 phase 完成度指标，而非结构性债务改善

**模式 C: "清理→扩张的往复"**
- Phase 9 做了大规模债务清理（ruff 下降 98%）
- Phase 10-16 转向能力扩张（知识脑、搜索产品面）
- 债务再次积累，在间歇性清理中循环

**模式 D: "敏感能力阻塞"**
- Apple/WeChat 连接器自 Phase 2 就识别为敏感能力
- 依赖 Safe Mesh + RBAC 安全基础设施
- 11+ 个 Phase 过去，安全基础设施仍未就绪

## 结构层 (Layer 3) — 产生这些模式的规则与反馈回路

**结构 1: "新能力优先于旧债务清理" 的激励结构**
- Phase 评估标准以"新能力交付"为主
- 债务清理不产生"可见进展"——没有新功能、没有新文档
- 激励结构天然偏向扩张而非收敛
- 反馈回路: 更多 Phase → 更多新能力 → health score 上升 → 更少动力清债

**结构 2: "健康分 = Phase 完成度" 的度量结构**
- 健康分的计算基于 task completion rate (124/126)
- 而非基于"核心债务是否被解决"
- 健康分可以 97 而核心债务 (D2, D3, SharedBrain) 全未解决
- 这个度量结构是一个信号过滤器——隐藏了真实的债务信号

**结构 3: "敏感能力 gate 独立于 Phase 节奏"**
- Apple/WeChat 的安全 gate (Safe Mesh + RBAC) 有自己的交付节奏
- Phase 迭代速度远快于安全基础设施交付
- 这两个时间尺度不同步，导致连接器长期阻塞
- 没有机制去"提前交付安全基础设施"——它只是"做完了再通知"的被动依赖

**结构 4: "SSOT 收敛 vs 影子增长" 的双重结构**
- `.omo` 治理体系强力收敛事实面（SSOT 铁律、四平面）
- 但同时新设计 (Task Center, Phase 设计文档) 略领先于 repo 实现
- 产生了"治理标准收紧 + 设计包扩张"的不对称结构

## 心智模型层 (Layer 4) — 使结构显得自然的内在信念

**信念 1: "新功能比清债更能证明系统在前进"**
- 持有者: Phase 规划者（隐含）
- 证据: 每次 Phase 主题都是"新功能", 清债只是附带的 wave
- 如何改变: 设立一个"债务清理 Phase"或"债务清理 KPI"——当 health score 的提升来源于 task completion 而非债务解决时，它反映的是假阳性信号

**信念 2: "健康分高 = 系统健康"**
- 持有者: 治理体系设计者
- 证据: health_score 从 90→97 时，核心债务 (D2, D3, SharedBrain) 状态未变
- 如何改变: 将 core debt items 纳入健康分计算权重

**信念 3: "敏感能力只能等安全基础设施就绪"**
- 持有者: 安全规划者
- 证据: Safe Mesh/RBAC 从未被排入 Phase 计划——它是个"永远在规划的依赖"
- 如何改变: 将 Safe Mesh 本身作为一个 Phase 的目标，而不是异步依赖

**信念 4: "Phase 是新能力的容器"**
- 持有者: 治理架构
- 证据: 16 个 Phase 中，0 个是纯债务清理 Phase
- 如何改变: 允许 Phase 被定义为"收敛 Phase"

# 系统原型匹配 (FindArchetype)

## 原型匹配: 两种原型的复合

### 原型 1: 负担转移 (Shifting the Burden) ⭐⭐⭐ 最高匹配度

**行为描述:**
系统有多项债务问题，每个问题都可以通过"症状解"（快速交付新功能掩盖债务）或"根本解"（花时间清理债务）来解决。症状解更便宜但产生依赖，根本解更昂贵但持久。系统反复选择了症状解。

**结构签名:**
- 症状解: 交付新 Phase → health score 上升 → 债务被掩盖
- 根本解: 花时间清理 D2/D3/SharedBrain → 没有新能力 → health score 可能不升反降
- 依赖形成: 每次延迟清债，清债成本上升（代码膨胀、重构难度增加）
- 根本能力萎缩: 随着时间推移，清理债务的专业技能/知识衰减

**证据:**
- D2/D3 跨越 12+ Phase 未解决
- SharedBrain 零测试从未排期
- 健康分 97 但核心债务全未解决

**典型干预 (来自 Senge):**
1. **有意识地投资根本解**——即使症状解更便宜
2. **给症状解设日落时限**——"如果 D2/D3 不在 Phase 17 解决，Phase 禁止进入新能力开发"
3. **显式追踪根本能力的增强/萎缩**——衡量"清债能力"是否在增长

### 原型 2: 目标漂移 (Drifting Goals) ⭐⭐⭐ 高度匹配

**行为描述:**
健康分/完成标准在不断地向下调整。原来的高门槛（"核心债务必须解决"）被悄悄替换成更容易达到的指标（"Phase 任务完成率"）。

**结构签名:**
- 实际债务状况 → 距理想状态有差距
- 理想状态被悄悄调整（从"消除核心债务"到"完成 Phase 任务"）
- 差距缩小了，但不是因为债务解决了，而是因为标准降低了
- 多次迭代后，"正常"门已被显著调低

**证据:**
- 健康分计算方式从"债务解决率"漂移到"任务完成率"
- D2/D3 从"必须解决"漂移到"下次一定"
- SharedBrain 从"必须加测试"漂移到"正式记录去留决策"

**典型干预:**
1. **锚定外部标准**——行业基准、第一性原理计算
2. **让漂移可见**——dashboard 显示历史目标线，展示侵蚀
3. **增强能力来缩小差距**——而不是降低标准

# 杠杆点分析 (FindLeverage)

## 系统目标

**当前隐含目标:** 完成 Phase 任务 → 提升健康分 → 推进系统演进
**期望行为:** 结构性债务持续减少，新能力建立在健康的基础上

## 候选干预措施 (按 Meadows 12 杠杆点排序)

| 干预 | Meadows 层级 | 可行性 | 有效杠杆 |
|------|:-----------:|:------:|:--------:|
| A: 健康分公式引入债务因子（D2/D3/SharedBrain 权重） | L6 信息流 | 高 | 中高 |
| B: 将 Safe Mesh + RBAC 设为一个独立 Phase 目标 | L5 规则 | 中 | 高 |
| C: 设立"收敛 Phase"原型（Phase 可以是纯债务清理） | L3 系统目标 | 中 | 高 |
| D: 转换心智模型——"健康 = 核心债务解决"而非"Phase 完成" | L2 范式 | 低 | 极高（长期） |
| E: 增加治理门禁：Phase GO 条件必须包含特定债务项被解决 | L5 规则 | 高 | 高 |
| F: 运行时间线仪表盘，展示债务迁移历史（D2 跨越了 XX 个 Phase） | L6 信息流 | 高 | 中 |
| G: D2 CI 环境容器化（具体技术方案） | L12 参数 | 高 | 低 |
| H: 正式记录 SharedBrain 去留决策 | L5 规则 | 高 | 中 |
| I: Orphaned_tasks 拆分为结构化 registry | L5 规则 | 中 | 中 |

## 最高有效杠杆选择

**选择 E + A 组合 (规则 + 信息流)**

**E — Phase GO 门禁规则:**
在 Phase 17 的 GO 条件中增加:
- D2 (CI E2E 测试环境) 必须已解决或已排期
- D3 (eu-pricing 独立测试) 必须已解决或已排期
- SharedBrain 必须有正式去留决策记录
- 否则 → No-Go

**A — 健康分公式重构:**
- 当前: task_completion_rate + phase_completion_status
- 改为: task_completion_rate × debt_weight
  - debt_weight = 1.0 if D2/D3/SharedBrain 均已解决
  - debt_weight = 0.85 if 部分解决
  - debt_weight = 0.7 if 全部未解决
  - 这样健康分 97 × 0.7 = 67.9 ——真实反映系统健康度

**附带低层干预 G (D2 容器化):**
- 在等待结构变化时，立即开始 D2 容器化技术方案
- 这不是替代结构变化，而是为结构变化争取时间

## 预期二阶效应

- E 实施后: Phase 17 可能延期，但债务被真正解决
- A 实施后: 健康分可能从 97 降至 70-85 区间（这是健康的——准确反映现实）
- 二阶效应: 团队可能抗议"健康分下降了"，需要前置沟通
- 三阶效应: 如果健康分不再能通过"完成 Phase 任务"轻易提升，债务清理会获得真正的优先级
