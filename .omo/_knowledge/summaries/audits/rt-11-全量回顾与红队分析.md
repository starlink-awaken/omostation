### 4.2 新增红队攻击 (3项)

**攻击 #11: 模块集成断裂 (严重度: 高)**

- **发现**: 新模块 (Memory Tree / TokenJuice / 集体复盘 / 生态市场) 均独立测试通过，但缺少跨模块集成测试
- **示例**: Memory Tree ingest→TokenJuice压缩→Router转发，从未合在一起跑过
- **影响**: 模块间接口可能不兼容
- **修复建议**: 创建 `phase13_integration_test.py`，做一条完整链路：用户输入 → 模型路由 → 记忆树查询 → TokenJuice压缩 → Agora路由 → Collab Task创建 → 共识标记 → 保鲜检查

**攻击 #12: SQLite 数据库碎片化 (严重度: 中)**

- **发现**: 系统现有至少8个独立的SQLite数据库文件：
  ```
  ~/.kos/kos.db         (KOS主数据)
  ~/.kos/usage.db       (成本追踪)
  ~/.kos/identity.db    (身份CA)
  ~/.kos/grants.db      (授权)
  ~/.kos/autopull.db    (自动拉取)
  ~/.kos/collective_reviews.db (集体复盘)
  ~/.kos/web_of_trust.db (信任网)
  ~/.kos/realtime.db    (实时协作)
  ```
- **影响**: 无法做跨库查询；备份要分别处理；复杂度增加
- **修复建议**: Phase 13考虑合并为统一的 `~/.kos/workspace.db`，用表前缀隔离

**攻击 #13: 无全量健康评分更新 (严重度: 中)**

- **发现**: STATE.md中的9维健康评分仍是Phase 7的旧分（75-90分区间），没有随着Phase 8-12的交付更新
- **影响**: 看不出实际健康度
- **修复建议**: 按Phase 12完成后的实际情况重新评分
