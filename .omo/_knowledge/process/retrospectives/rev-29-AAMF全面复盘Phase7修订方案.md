## 四、Phase 7 修订方案

### 原始方案评估

| 原始任务 | 评估 | 建议 |
|---------|------|------|
| 7.1 元模型变更控制 (6h) | 超前：当前 24 约束稳定，元模型变更不频繁 | **降级为脚本化**，不做版本化 |
| 7.2 宪法修订自动化 (6h) | 有价值：宪法手动修订容易遗漏 | **保留**，简化实现 |
| 7.3 架构模式识别 (8h) | 超前：仅 31 条日志 + 3 个熵数据点，无模式可学 | **删除**，等运行 3 个月再说 |
| 7.4 自进化验证 (6h) | 有价值 | **保留**，缩小范围 |
| 7.5 终极自指测试 (4h) | 已有基础：hotswap 支持 governance-system | **保留**，简化为一次性验证 |
| 7.6 最终复盘 (4h) | 文档为重 | **保留** |

### 修订后方案

```
Phase 7 核心目标: 完成 L4 自我层，让治理体系具备自我修订能力
精简后: 原 34h → 12h，聚焦真正有价值的部分
```

| ID | 任务 | 工时 | 产出 | 说明 |
|----|------|------|------|------|
| **7.1** | 宪法修订流水线 | 4h | `arcnode-amend` CLI | LLM提议→人类确认→自动修订宪法文档+git commit |
| **7.2** | 自指热插拔验证 | 3h | 端到端测试 | `agora hotswap governance-system --new-yaml v2` 闭环 |
| **7.3** | 3 个月后模式识别预留 | 0h | 占位文档 | 等治理日志积累到 200+ 条再启动 |
| **7.4** | 最终审计报告 | 3h | #30 文档 | 全体系验收 + Phase 7 复盘 |
| **7.5** | schema.py→宪法自动同步 | 2h | `arcnode-sync-constitution` | 检测 schema.py 约束与宪法文档的差异并修复 |

### 修订后执行顺序

```
Step 1: 7.5 schema→宪法自动同步 (防漂移)
Step 2: 7.1 宪法修订流水线 (LLM提议→人类确认→自动修订)
Step 3: 7.2 自指热插拔验证 (闭环测试)
Step 4: 7.4 最终审计报告
Step 5: 7.3 模式识别预留文档 (不执行，只标记)
```

### 验收标准 (修订后)

```bash
# 7.5 schema→宪法同步
arcnode-sync-constitution --check
→ constraints.md 与 schema.py 一致 ✅

# 7.1 宪法修订
arcnode-amend --proposal "Add constraint T8 for EVOLVER must_have"
→ 生成修订提案 → 人工确认 → 自动更新 CONSTITUTION.md + git commit

# 7.2 自指热插拔
agora hotswap governance-system --dry-run
→ 7 步骤输出，含 governance-system 自身替换步骤 ✅
```

## 五、关键决策

| # | 决策 | 选项 | 选择 | 理由 |
|---|------|------|------|------|
| 1 | 元模型变更控制做不做 | 版本化 / 脚本化 / 不做 | **脚本化** — schema.py 编辑 + `arcnode-sync-constitution` 即可 | 24 约束稳定，版本化过度设计 |
| 2 | 模式识别做不做 | 做 / 等3个月 / 不做 | **等 3 个月** — 数据量不足 | 31 条日志无模式可学 |
| 3 | 自指测试的深度 | 完整替换 / 仅 dry-run | **完整替换** — 已有 hotswap 基础，风险低 | hotswap 的 R2+R3+R10 门禁足以保障 |
| 4 | 宪法修订的人工确认 | 必须 / 可选 | **必须** — 宪法修订涉及治理规则 | 宪法是治理根基，不能自动修改 |
| 5 | Phase 7 后做什么 | L4 自我层 / 收工 / 用户场景 | **收工** — AAMF 治理体系完成 | 原始蓝图 7 个 Phase 全部覆盖 |

## 六、AAMF 终点定义

### 完成的标志

```
宪法:   8 章, 24 约束, 自动同步
节点:   26 注册, 含 governance-system 自身
脚本:   14 CLI, 覆盖注册/更新/热插拔/进化/视图/审计
日志:   SHA256 链式校验, 可追溯
Cron:   7 道定期任务, 自动维护
视图:   C4 四层 + Archimate 三层 + 健康仪表盘
自指:   governance-system 可替换自身
宪法修订: LLM 提议 → 人类确认 → 自动落盘
```

```

❌ Eidos 元模型同步  — AAMF 独立于 Eidos 运行
❌ 多机器拓扑视图    — 属于 X2 但无实际需求
❌ 自动化部署 pipeline — AAMF 是治理体系，不是 CI/CD
```
