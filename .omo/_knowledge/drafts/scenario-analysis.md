## Top 10 用户旅程验证

### A1 — 深度研究一个技术主题

**当前可行度**: ✅ 部分可行 (workspace research 已实现基础版)

**用户旅程**:
```
用户输入: workspace research "transformer attention variants"
    ↓
[当前] 带着进度条研究执行 (rich Progress)
    ↓
[当前] 结果显示在终端 (rich Panel + Markdown)
    ↓
[当前] 自动保存到 SQLite (~/.workspace/data.db)
    ↓
[缺失] 用户可以浏览所有研究历史的可视化 UI
```

**断点**: 结果在 CLI 中好看但在 Web 上看不到。Dashboard 已有 research API 和 HTML 卡片，但 Agora Dashboard 和 workspace CLI 是两套系统。
**用户当前必须**: `workspace research --list` 看列表, `workspace research --open N` 看全文。Dashboard 需要 `cd agora && uvicorn run`。
**修复**: workspace dashboard 命令一键启动 Web UI。`workspace open-dashboard` 打开浏览器。

---

### A2 — 后续追问

**当前可行度**: ✅ 部分可行 (workspace research --ask 已实现)

**用户旅程**:
```
workspace research --ask 1 "what about sparse attention?"
    ↓
[当前] 追问自动关联到原研究记录 (follow_ups JSON)
    ↓
[当前] workspace research --open 1 显示追问对话
    ↓
[缺失] 追问结果通知 (如果有新研究结果)
```

**断点**: 追问是基于已有研究的推测性回答，没有真正调用 minerva 重新研究。追问和原研究在一个记录里但用户不一定知道 `--open 1` 能看到。
**修复**: workspace research --ask 应调用 minerva research 获取真实结果。追问完成后提示 "新发现已添加到研究 #1"。

---

### B2 — 知识搜索

**当前可行度**: ✅ 部分可行 (workspace research --list)

**用户旅程**:
```
workspace research --list
    ↓
[当前] 显示最近研究列表 (rich Table)
    ↓
[缺失] 全库搜索 (跨所有研究内容的全文搜索)
    ↓
[缺失] 按标签、时间、关键词过滤
```

**断点**: 只能按 ID 打开已知研究或浏览最近列表。不能搜索研究全文内容。
**修复**: 加 `workspace research search <keyword>`。SQLite 的全文搜索 (FTS5) 即可。

---

### D1 — 系统健康

**当前可行度**: ✅ 部分可行 (workspace status 已实现)

**用户旅程**:
```
workspace status
    ↓
[当前] rich Table 显示服务状态
    ↓
[缺失] 自动检测服务是否实际响应 (不只是 CLI 是否存在)
    ↓
[缺失] 发现异常时报警
```

**断点**: CLI 只检测二进制是否存在，不检测服务是否真正运行。Agora 可能在跑但端口不通。
**修复**: workspace status 应做 HTTP health check: `curl localhost:7430/health`。

---

### C7 — 结果通知

**当前可行度**: ✅ 部分可行

**用户旅程**:
```
workspace research "long topic" (> 60秒)
    ↓
[当前] 用户盯着进度条等 (rich Progress 有 TimeElapsed)
    ↓
[缺失] 研究完成后弹通知 (macOS notification)
    ↓
[缺失] 用户不在终端前就错过了完成消息
```

**断点**: CLI 研究完成后就只能终端看到。如果用户走开了，看不到完成通知。
**修复**: 加 `terminal-notifier` 或 `osascript -e 'display notification'` 发送 macOS 系统通知。

---

### E3 — 全量测试

**当前可行度**: ✅ 部分可行 (make test)

**用户旅程**:
```
cd workspace && make test
    ↓
[当前] 逐个项目运行测试，输出到终端
    ↓
[缺失] 测试结果聚合 (哪些过哪些不过一目了然)
    ↓
[缺失] 失败的测试有错误详情和修复建议
```

**断点**: `make test` 输出原文是各项目原生的，没有汇总。失败项目需要手动翻输出。
**修复**: `make test` 用 rich.Table 汇总结果显示。失败时给出简化的错误摘要。

---

### F1 — 每日研究简报

**当前可行度**: ✅ 部分可行

**用户旅程**:
```
workspace research --list
    ↓
[当前] 看到最近研究的列表
    ↓
[缺失] 自动整理的每日简报，包含摘要、关键发现、趋势
    ↓
[缺失] 推送 (每天早上自动运行)
```

**断点**: 没有自动化生成简报的命令。手动看列表不能替代简报。
**修复**: 加 `workspace daily` 命令，聚合昨天到今天的研究，自动生成摘要。
