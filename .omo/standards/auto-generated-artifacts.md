# 自动生成产物治理标准

> 作用域: 根仓库 `.omo/`、`bin/`、`docs/` 中由 post-commit hook / CI / Agent 自动生成的文件。
> 权威来源: `.git/hooks/post-commit` (调用 `projects/ecos/src/ecos/ssot/tools/mof-extract.py` 等工具)。

## 原则

1. **自动生成 ≠ 免审**: hook 生成的文件进入工作区后, 下一次 commit 必须由人类或 Agent 显式 review。
2. **显式标记**: 所有自动生成文件必须在顶部包含 `AUTO-GENERATED by <tool>` 标记。
3. **SSOT 不漂移**: 自动生成文件不得复制 `.omo/_truth/` 或 `.omo/state/` 中的事实, 只能引用相对路径。
4. **可重现**: 删除后必须能由同一命令重新生成。

## 处理流程

```
commit → post-commit hook 异步生成 → working tree 出现新文件
  → 下次 commit 前: git status 检查
  → review 内容 / 加 AUTO-GENERATED 标记
  → stage + commit (带明确 message)
```

## 禁止行为

- 未经 review 直接 `git add -A` 提交自动生成产物。
- 在自动生成文件中手工维护与真源冲突的事实。
- 将临时 `.tmp` / `.auto.yaml` 产物提交到仓库。

