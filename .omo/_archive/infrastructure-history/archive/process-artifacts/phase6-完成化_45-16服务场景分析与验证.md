# 16服务场景分析与真实验证

## 一、服务场景全景图

### 按用户旅程归类

```
用户旅程入口                    → 中游路由               → 下游执行

"帮我研究 X" (研究)              → pallas → agora         → minerva → kos/eidos
"存一下这篇文章" (知识)          → kronos                 → eidos → iris(vault/WPS)
"有哪些工具能做 X" (发现)        → forge                  → agora → 目标服务
"系统健康吗" (运维)              → agora → bos-daemon     → ssot-kernel
"分析这个项目" (分析)            → codeanalyze             → eidos → kos
"配置新服务" (治理)              → bos-skill-cli           → forge/ssot-kernel
"调度复杂任务" (编排)            → agentmesh               → metaos → 全链路
```

## 二、5个端到端验证场景

### 场景1: 知识摄取全链路 [Knowledge Pipeline]
**目标**: 验证 kronos → eidos → iris 完整走通

```
Step 1: kronos 抓取一篇文章
  → 确认 5 层抓取引擎可用
  → 输出结构化内容

Step 2: eidos schema 验证
  → 将内容映射为 KnowledgeCard
  → 通过 eidos validate 校验

Step 3: iris 分发
  → 确认 iris MCP 工具列表
  → 确认 connector 列表
```

### 场景2: 研究发现全链路 [Research]
**目标**: 验证 pallas → agora → minerva → kos 完整走通

```
Step 1: pallas CLI 调用
  → pallas derive/check/match 命令可用
  → 委派到 ontoderive/agora

Step 2: minerva 研究
  → 5 个 MCP 工具全部注册
  → research_now 可执行

Step 3: kos 知识存储
  → CLI 命令可执行
  → MCP 入口可调用
```

### 场景3: 工具发现全链路 [Tool Discovery]
**目标**: 验证 forge → agora → 目标服务注册路由

```
Step 1: forge 工具目录
  → tools-registry.json 可读
  → capability-index 索引完整

Step 2: agora 服务/路由
  → 16个服务全部注册
  → 路由全部对齐

Step 3: 特定服务可达
  → 选择一个下游服务
  → 通过 agora 路由确认
```

### 场景4: 系统运维全链路 [Operations]
**目标**: 验证 agora → bos-daemon → ssot-kernel 状态可查

```
Step 1: agora 健康检查
  → agora web dashboard 可访问
  → agora MCP 工具正常

Step 2: bos-daemon 脉冲
  → port 7420 监听中
  → 健康检查返回

Step 3: ssot-kernel 配置
  → 50/50 测试
  → 核心 pattern 可用
```

### 场景5: 编排与分析全链路 [Orchestration + Analysis]
**目标**: 验证 agentmesh → metaos → codeanalyze → kos 协作

```
Step 1: codeanalyze 分析
  → 12个CLI命令可用
  → 图谱导出可用

Step 2: agentmesh 编排
  → build 通过
  → 核心包可导入

Step 3: metaos 协调
  → 39/39 测试通过
  → register_agora 可用
```
