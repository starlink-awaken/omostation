## 一、T0-T3 分级体系

### T0 — 主力 SSD（随机读写 ≤10μs）

| 盘 | 容量 | 归属 | 内容 | 不可替代？ |
|----|------|------|------|-----------|
| 外接 4TB SSD (`/Volumes/Work`, `/Volumes/Model`) | 4TB | **Mac mini** | Docker 数据、Ollama 小模型、全量代码、KOS 知识库 | ✅ Mac mini 核心 |
| MBP 内置 SSD | 1TB+ | **MBP** | 系统、当前活跃项目 | ⚠️ 可远程 |
| 三星 SSD (`/Volumes/DevDrive`) | 1TB | **MBP 随身** | 离线模型、出差项目文件 | ✅ 出差必备 |
| Y7000P 内置 NVMe | 4TB | **Y7000P** | 游戏、源码镜像 | ❌ 可重装 |
| 希捷 SSD | 2TB | 备用 | 待分配 | — |

### T1 — 温数据 HDD（顺序 ~200MB/s）

| 盘 | 容量 | 归属 | 内容 |
|----|------|------|------|
| 东芝 4TB SAS（转接） | 4TB | 闲鱼待出 | 大文件归档、ISO、模型权重备 |

### T2 — 冷备份 SMR HDD（~130MB/s）

| 盘 | 容量 | 归属 | 内容 |
|----|------|------|------|
| WD_BLACK P10 | 2TB | Y7000P 挂载 | Time Machine 目标 ×2 |

## 二、数据保护策略

### 三二一原则

```
每份数据：    3 份副本   ×   2 种介质   ×   1 份异地
```

| 数据类型 | 副本1 | 副本2 | 副本3 (异地) |
|---------|-------|-------|-------------|
| 源码仓库 | Mac mini SSD | GitHub | MBP 本地 |
| Ollama 模型 | Mac mini SSD | Y7000P 内置 | — |
| 照片/知识库 | Mac mini SSD | Google One | — |
| 系统全量 | Mac mini SSD | WD_BLACK (Time Machine) | — |

### 备份流向

```
数据源                   目标
──────────────────────────────────────────────────
Mac mini (4TB SSD)  ──→ smb://100.x.x.3/Mac_TimeMachine
                          → WD_BLACK P10 (MacMini_TM.sparsebundle)

MBP (内置 SSD)      ──→ smb://100.x.x.3/Mac_TimeMachine
                          → WD_BLACK P10 (MBP_TM.sparsebundle)

Mac mini (核心文件)  ──→ Google Drive (Obsidian/配置快照)

源码                ──→ GitHub (starlink-awaken 组织)
```

## 三、Time Machine 配置

### Mac mini

```bash
# 挂载 Y7000P SMB
open smb://100.x.x.3/Mac_TimeMachine

# 创建 Sparse Bundle (1TB 上限)
hdiutil create -size 1t \
  -type SPARSEBUNDLE \
  -fs APFS \
  -volname MacMini_Backup \
  /Volumes/Mac_TimeMachine/MacMini_TM.sparsebundle

# 指定备份目标
sudo tmutil setdestination '/Volumes/MacMini_Backup'
tmutil destinationinfo
```

### MBP M5 Max

```bash
# 挂载同一 SMB
open smb://100.x.x.3/Mac_TimeMachine

# Sparse Bundle (500GB 上限)
hdiutil create -size 500g \
  -type SPARSEBUNDLE \
  -fs APFS \
  -volname MBP_Backup \
  /Volumes/Mac_TimeMachine/MBP_TM.sparsebundle

sudo tmutil setdestination '/Volumes/MBP_Backup'
```

### Automator 自动挂载

创建应用程序工作流：
1. 获取指定的服务器 → `smb://100.x.x.3/Mac_TimeMachine`
2. 连接到服务器
3. 打开访达项目 → `MacMini_TM.sparsebundle` / `MBP_TM.sparsebundle`
4. 设为登录项

## 四、WD_BLACK P10 注意事项

| 属性 | 说明 |
|------|------|
| 真实身份 | 2.5" 机械硬盘 (SMR)，非 SSD |
| 极限速度 | ~130MB/s 顺序读写 |
| 随机写入 | ❌ 极差（SMR 通病） |
| 正确定位 | **仅做 Time Machine 冷备份，不做任何活跃数据盘** |
| 是否适合跑模型 | ❌ 绝对不行 |
| 是否做编译盘 | ❌ 绝对不行 |
